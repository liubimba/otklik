# Скриншоты экранов для секции «Как это работает»

Сейчас здесь **заглушки** — схематичные вайрфреймы, а не настоящие скриншоты.
Их нужно заменить реальными снимками приложения.

## Как заменить

Просто положите файл с тем же именем. Ничего в коде править не нужно —
пути перечислены в `src/lib/content.ts` (поле `shot` у каждого шага).

| Файл | Что должно быть на скриншоте |
|---|---|
| `step-1-auth.{light,dark}.svg` | Вход в hh.ru: открытое окно авторизации |
| `step-2-settings.{light,dark}.svg` | Настройки: резюме, стиль письма, список LLM-провайдеров |
| `step-3-search.{light,dark}.svg` | Очередь вакансий: карточки со статусами |
| `step-4-review.{light,dark}.svg` | Разбор письма: текст письма + AI-чат |
| `step-5-apply.{light,dark}.svg` | Отправленные отклики в очереди |

## Требования

- **Две версии на шаг** — `.light.svg` и `.dark.svg`. Лендинг показывает нужную
  по теме страницы, так что скриншот в тёмной теме на белом фоне будет выглядеть
  чужеродно. Снимайте один и тот же экран дважды, переключив тему в приложении.
- **Размер 1248×765** (или любой с тем же соотношением 1.63:1). Другая пропорция
  сломает вёрстку: размеры прописаны в `content.ts` и передаются в `next/image`.
- **Можно PNG/WebP вместо SVG.** Тогда поменяйте расширение в поле `shot`
  в `src/lib/content.ts` — например `/screens/step-1-auth.light.png`.
- Не обрезайте окно приложения: рамка мокапа на лендинге рисуется вокруг
  скриншота, поэтому свои тени и скругления добавлять не надо.
